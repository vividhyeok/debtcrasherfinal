# PATCH 09 - 세션 히스토리

## 1. 세션 파일 구조
이번 패치에서는 Agent View 대화 상태를 워크스페이스의 `.ai-sessions/` 아래 JSON 파일로 저장하도록 바꿨다. 파일명은 `YYYY-MM-DD-HH-mm-첫요청요약.json` 형식이며, 첫 요청 30자를 세션 제목으로도 함께 보관한다.

각 세션 파일에는 `title`, `startedAt`, `updatedAt`, `messages`가 들어간다. `messages`는 순서를 그대로 유지하며, 모든 항목에 `role`, `type`, `content`, `timestamp`를 저장한다. planning 메시지는 `summary`, `assumptions`, `questions`, `userChoices`를 추가로 저장하고, result 메시지는 생성 파일 목록과 검증 요약, 실행 안내, 수동 재검증 관련 상태를 저장한다. status 메시지는 본문과 별개로 `phaseLabel`만 구조화해서 남긴다.

민감한 설정값이나 API 키는 세션 파일에 저장하지 않는다. `.ai-sessions/`는 저장 시점에 `.gitignore`에 자동 추가되고, 세션 파일 수가 50개를 넘으면 가장 오래된 파일부터 자동 삭제한다.

## 2. 두 모드 레이아웃과 토글 동작
Agent View 상단에는 `새 채팅`과 `작업 기록` 두 모드 토글이 추가되었다. 기본 진입은 `새 채팅`이며, 오늘 생성된 세션 파일이 있으면 그 대화를 자동 복원하고 상단에 `이전 세션을 불러왔습니다.` 배너를 보여준다.

`작업 기록` 모드에서는 `.ai-sessions/` 안의 세션 목록을 한 줄씩 보여준다. 각 항목은 `세션 제목 · 날짜 시각` 형식으로 표시되며, 클릭하면 오른쪽 상세 영역이 아니라 Agent View 내부의 읽기 전용 대화 화면으로 세션 전체를 재구성한다. 이 화면은 planning 카드, 상태 메시지, 구현 결과 메시지를 원래 순서대로 다시 보여주고, 하단에는 `이 세션 이어서 개발` 버튼이 붙는다.

토글 버튼 스타일은 `var(--vscode-button-*)` 계열 변수만 사용해 VS Code 테마에 맞춰 동작한다.

## 3. resume 흐름
resume 진입점은 두 가지다. 첫 번째는 `작업 기록` 모드에서 `이 세션 이어서 개발` 버튼을 누르는 경우다. 이때 Agent View는 `새 채팅` 모드로 돌아가고, 입력창에 `이전 세션 이어서: [세션 제목]`을 미리 채운다.

두 번째는 사용자가 새 요청 자체에 `이어서`, `계속`, `resume`, `continue`, `다시`, `중단된` 중 하나를 포함해 보내는 경우다. 이때는 가장 최근의 관련 세션을 찾아 planning 전에 resume 컨텍스트로 주입한다.

두 경우 모두 planning 프롬프트에는 `이전 세션 요약: [마지막 사용자 요청] → [확정된 판단] → [무엇을 만들었는지 또는 무엇이 실패했는지]` 형식의 문장이 함께 들어간다. 그래서 이전 세션에서 이미 답한 판단은 다시 질문하지 않고, 새 요청에 필요한 판단만 남기도록 유도한다.

## 4. 새로고침 시 복원되는 것과 복원되지 않는 것
웹뷰를 닫거나 VS Code를 다시 열어도 다음 정보는 복원된다: 오늘 마지막 세션의 전체 메시지 순서, planning 카드 내용, 사용자가 선택한 옵션, 구현 결과 요약, 자동 검증 결과, 작업 기록 목록.

반대로 복원되지 않는 것은 현재 진행 중이던 네트워크 요청 자체와 입력창에 아직 전송하지 않은 임시 텍스트다. 즉 저장된 대화 상태는 다시 볼 수 있지만, 중간에 끊긴 planning/implementation 호출은 자동으로 이어서 실행되지 않는다.
