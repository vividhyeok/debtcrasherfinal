# PATCH 08 - 튜토리얼 생성 품질 보강

## Fix 1: 출력 포맷 검증 추가
기존에는 LLM이 markdown을 반환하면 그대로 저장해서, 섹션이 빠지거나 문서가 지나치게 짧아도 Step View가 성공으로 처리했습니다. 이번 수정에서는 `generateTutorial()` 직후 결과를 검사하고, 필수 섹션이 빠졌으면 파일을 저장하지 않고 즉시 오류를 반환하도록 바꿨습니다.

이 수정으로 해결한 문제는 다음과 같습니다.
- 필수 섹션이 없는 튜토리얼이 `.ai-tutorials`에 저장되던 문제
- 비교표나 연결 구조가 빠져도 성공처럼 보이던 문제
- 사용자가 잘못 생성된 문서를 지우고 다시 선택해야 하던 문제

검증 규칙은 아래와 같습니다.
- 선택한 step 수만큼 `결정한 것` 문구가 존재해야 함
- markdown 표가 최소 1개 이상 존재해야 함
- 선택한 step 수만큼 `이 선택이 이후 결정에 미친 영향` 문구가 존재해야 함
- 2개 이상 step을 선택한 경우 `판단들의 연결 구조`가 존재해야 함
- 2개 이상 step을 선택한 경우 `내 판단 패턴 분석`이 존재해야 함
- 전체 문서 길이는 선택 step 하나당 최소 400자 이상이어야 함

검증 실패 시 동작은 이렇게 바뀌었습니다.
- markdown 파일을 저장하지 않음
- 누락된 섹션 이름과 현재 상태를 포함한 구체적인 오류 메시지를 반환함
- Step View의 기존 inline error banner에 그대로 표시됨
- 사용자의 step 선택 상태는 유지되어 바로 재시도 가능함

## Fix 2: 토큰 예산 동적 계산
기존에는 선택 step 수와 상관없이 항상 `1800` 토큰으로 튜토리얼 생성을 요청했습니다. 이번 수정에서는 선택한 step 수에 따라 토큰 예산을 늘려, 여러 step을 동시에 선택했을 때 문서가 과도하게 짧아지는 문제를 줄였습니다.

토큰 예산 표는 다음과 같습니다.

| 선택한 step 수 | 토큰 예산 |
|---|---:|
| 1개 | 1200 |
| 2개 | 2000 |
| 3개 | 2800 |
| 4개 이상 | 3600 |

## Fix 3: 프로젝트 현재 상태 컨텍스트 추가
기존 튜토리얼 생성 입력에는 선택한 decision log만 들어가고, 현재 프로젝트가 어떤 기준 문서를 쓰는지나 최근 구현 결과가 무엇이었는지 정보가 없었습니다. 이번 수정에서는 `AGENT.md`의 최신 압축 캐시 내용과 최근 구현 요약을 함께 읽어 `프로젝트 현재 상태` 블록으로 user prompt 앞부분에 추가했습니다.

이 수정으로 해결한 문제는 다음과 같습니다.
- 튜토리얼이 현재 프로젝트 제약과 분리된 일반론으로 흘러가던 문제
- 최근 구현 맥락이 빠져서 판단 결과가 어떤 산출물로 이어졌는지 약하게 설명되던 문제
- decision log만 보고 문서를 써서 현재 기준 문서와 어긋나는 표현이 나올 가능성
